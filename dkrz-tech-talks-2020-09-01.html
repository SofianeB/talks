    one would need to log into the HPC cluster, use the command-line to reserve computing resources and set up some SSH tunneling.

Spawner options

Overview



Once you login, there are two tabs where you can choose between preset or advanced options form.
These are two different spawners to customize your session: this means what are the resources that will be allocated to your jupyter notebook (memory, CPUs, time. etc).

Preset options form


Choose from the list Available profiles
  1. in the preset form we already configured some profiles with the most important Slurm options (cores, memory and time). In the right side you see the available profiles.
    Depending on what you want to do you just pick the one you think it fits your requirements.
    • shared partition; small development/testing projects </small>
    • prepost: as before, but allowing access outside of mistral (wget or similar)

The other fields are not mandatory (reservation and qos).

Advanced options form (1)

for advanced users we have the advanced spawner options. this options form allows you to customize many parameters.
for example, you can:

  • choose which jupyter interface to start with (classic or lab)
  • modify slurm parameters
  • rename the log file

Advanced options form (2)

This table just shows the parameters and their descriptions, this available in the documentation.

Advanced options form (3)

  • most of the errors that you might see with the advanced spaner are due to a misconfiguration of the slurm parameters.
    in this case you will see something like this: sbatch error.
    for some fields there is an information about the origin of the error (cpu, memory).
  • For the other fields I suggest that you take a look first at the Mistral configuration page. there you will see the available partitions with their resources.

Named servers


  • Named servers allow you to have more than one server running in the same time
  • Currently 2 allowed: preset and advanced
  • Extendable --> HLRE 4 --> more

this is not a new feature for our deployment but a new feature from the jupyterhub package itself.

Stop your server

  • loging out or closing your browser won’t stop your session automatically.
  • your notebook is running as Slurm job, if you finish your work you need to explictly stop the server, otherwise it will continue consuming the allocated resource.
  • as you can see in the animation, stoping the server depends on how you start it: preset or advanced ans also the interface …

Bring your own environment (1)

  • Conda 
    
% mkdir $HOME/kernels
% conda create --prefix $HOME/kernels/tensorflow ipykernel python=3.x
% source activate $HOME/kernels/tensorflow
% python -m ipykernel install --user --name tensorflow --display-name="tensorflow"
% conda deactivate
  • Virtualenv 
    % python -m pip install --user virtualenv
% python -m virtualenv --system-site-packages /path/to/new-kernel
% source /path/to/new-kernel/bin/activate
% pip install ipykernel
% python -m ipykernel install --user --name new-kernel --display-name="new-kernel"
  1. If you have your own Python environment, depending on how it is installed, it may just show up in the list of kernels you can use.
    If not, use the following procedure (also documented in the technical documentation) to enable a custom kernel based on a Conda/virtualenv environment. You need to restart your seesion to enable the new changes.
  2. To use conda binary you need to load one of our system python modules (python3/unstable)
  3. best practices –> where to create the new env.

Debugging

  • preset spawner
    • jupyterhub_slurmspawner_preset_{slurm_job_id}.log
  • advanced spawner
    • default: jupyterhub_slurmspawner_advanced_{slurm_job_id}.log
    • customized: you_name_it.log

  1. When you launch Jupyter notebooks on mistral. Logs are saved files in your HOME directory. These logs can be very helpful when it comes to debugging issues with Jupyter, your custom kernels, or your Python environment.
  2. The name of the log file is related to the spawner options form you used: preset or advanced.
  3. support@dkrz.de
  4. you can control the verbosity or logging level by modyfing the jupyter_notebook_config file in your home. c.Application.log_level.

Disable:
